## 🔍 Detail Analysis

### 1. NULL_RETURNS (High Risk)
*   **Description:** Function `findPaymentInDB()` might return `null` when a transaction reference is missing or invalid.
*   **Impact:** A service crash occurs when internal code tries to access `payment.status` without checking for null.
*   **Evidence:**
    ```typescript
    const payment = this.findPaymentInDB(params.vnp_TxnRef);
    console.log(`Checking status for payment: ${payment.status}`); // <--- CRASH HERE
    ```

### 2. SECURITY_TAINT (Web Security Vulnerability)
*   **Description:** The variable `vnp_Amount` is received directly from a web request (VNPay callback) and used in financial calculations without sufficient validation of its source or integrity (Taint tracking).
*   **Impact:** Potential **Data Tampering**. An attacker could theoretically spoof the callback to credit their wallet with a different amount.
*   **Recommendation:** Use strict HMAC verification (already implemented in `vnpay.service.ts` but needs to be enforce in the entry controller).

### 3. INSECURE_RANDOM (Quality)
*   **Description:** Using `Math.random()` for transaction references is not cryptographically secure.
*   **Recommendation:** Use `crypto.randomBytes()`.
